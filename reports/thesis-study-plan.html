Thesis Study Plan

Interactive single-page study website for master thesis preparation covering cryptography, ZKP, TEE, and privacy systems

Vanilla JS HTML/CSS KaTeX localStorage Vercel No Build Step

The Challenge

Preparing a master thesis on anonymous credentials requires studying across 6+ dense technical domains simultaneously.

The thesis spans anonymous credentials, confidential transactions, trusted execution environments, private payments, ZK proof systems, and Rust implementation. Each chapter has academic papers, video lectures, blog posts, and exercises. Tracking progress across hundreds of resources with different depth levels is overwhelming without structure.

This project creates a tailored study dashboard that organizes all thesis preparation material by chapter, provides two depth levels (intuitive overview vs. technical deep dive), tracks completion progress via checkboxes persisted in localStorage, and includes productivity tools like a Pomodoro timer and global concept search.

Why It Matters

  • Chapter-based organization: 6 thesis chapters + a Rust track + a ZK Deep Dive, each as a tab with color-coded content and per-chapter study guides.
  • Dual-view learning: every topic has an "Intuitive" view (analogies, diagrams, high-level concepts) and a "Technical" view (math notation via KaTeX, protocol specifications, proof sketches).
  • Progress persistence: checkbox states saved to localStorage, with a global progress bar showing overall completion percentage across all chapters.
  • Zero dependencies: pure HTML/CSS/JS with no framework and no build step. Open index.html in a browser or deploy to Vercel.

Key Insights

Zero-Dependency Architecture

No npm, no bundler, no framework. The entire application is plain HTML/CSS/JS files loaded directly by the browser. KaTeX is the only external dependency (loaded via CDN). This means zero build time, instant deployment to Vercel, and no version conflicts. The tradeoff is more boilerplate in the JS data files, but for a study tool that is primarily content, this is the right call.

Dual-View Content Model

Every study guide topic ships with two renderings. The "Intuitive" view uses analogies, simplified diagrams, and plain-language explanations. The "Technical" view shows the actual math (rendered via KaTeX), protocol specifications, and proof sketches. A global toggle in the nav bar switches all content simultaneously, with the preference saved to localStorage.

Data-Driven Content

All study content lives in separate .js data files (30+ files), not inline in the HTML. Each file exports structured objects with titles, descriptions, resources, and exercises. The study-guide.js renderer consumes these data files and generates the DOM. This separation makes it easy to add new chapters or update content without touching the app logic.

Productivity Integration

The Pomodoro timer (25 min work / 5 min break) is built into the nav bar with start/pause/reset controls and audio notifications. Combined with the progress bar showing completion percentage, it creates a focused study environment where the user can track both time spent and material covered.
