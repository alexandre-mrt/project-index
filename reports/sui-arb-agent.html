Sui Arb Agent

Autonomous DeFi arbitrage agent detecting price discrepancies between DeepBook v3 and Cetus on Sui

TypeScript Sui Move DeepBook v3 Cetus CLMM Bun PTBs Vitest

The Challenge

Multiple DEXs on Sui (DeepBook v3 as a central limit order book, Cetus as a concentrated liquidity AMM) regularly exhibit price discrepancies for the same token pairs. Manual arbitrage is too slow: by the time a human spots and executes, the opportunity vanishes. An autonomous agent that continuously scans prices and executes atomic cross-DEX swaps can capture these spreads in milliseconds.

Core problem: Price feeds across Sui DEXs diverge constantly, but opportunities close within seconds. Only an automated system with real-time monitoring and atomic execution can reliably capture them.

Why It Matters

  • Market efficiency: Arbitrage bots are essential price correctors that keep DEX prices aligned across the Sui ecosystem
  • MEV on Sui: Unlike EVM chains, Sui's object-centric model and Programmable Transaction Blocks enable a different MEV landscape with atomic multi-step transactions
  • Atomic execution: PTBs guarantee all-or-nothing execution, eliminating partial fill risk that plagues multi-tx arbitrage on other chains
  • Flash loan support: DeepBook v3 flash loans enable capital-free arbitrage, lowering the barrier to entry

How It Works

Key flow: Event listener detects price changes on DeepBook/Cetus → Price aggregator computes cross-DEX spread → Strategy validates profitability after gas → PTB builder constructs atomic swap (buy cheap, sell expensive, validate on-chain) → Executor signs and submits in a single transaction.

Architecture

On-chain (Sui Move)

ModulePurpose
vault.moveShared vault with Bag-based generic coin storage. AdminCap/KeeperCap capability-gated access for deposits and withdrawals.
strategy.moveOn-chain strategy config (min spread, max trade size) with circuit breaker. Admin-only updates via shared object.
validator.moveProfitability validation: spread calculation in bps, profit estimation with gas cost deduction, trade direction logic. Uses u128 to prevent overflow.
events.moveTrade event structs (TradeExecuted, ArbitrageFailed) for transparent on-chain logging with timestamps.

Off-chain (TypeScript/Bun)

ModulePurpose
prices/deepbook.tsDeepBook v3 price feed via devInspectTransactionBlock. Reads mid price and level2 order book (best bid/ask). Custom BCS parsing for u64 and vector<u64>.
prices/cetus.tsCetus CLMM price feed. Reads pool object, extracts current_sqrt_price (X64 fixed-point), converts to price via integer-only math. Newton's method for bigint sqrt.
prices/aggregator.tsCross-DEX spread calculation in basis points. Compares bid/ask across both DEXs, determines trade direction, estimates gross profit.
strategy/arbitrage.tsOpportunity detection with gas-aware profitability check. Conservative sizing at 80% of max to account for slippage.
executor/builder.tsPTB construction for direct and flash-loan arbitrage. Composes buy/sell/validate/emit in a single transaction block.
executor/executor.tsTransaction signing, submission, and result parsing. Dry-run support for pre-flight validation. Gas computation from effects.
listener/events.tsDual-mode monitoring: WebSocket event subscriptions (real-time) + polling fallback (reliability). Debounced to prevent redundant checks.

Key Insights

Atomic swaps via PTBs. Sui's Programmable Transaction Blocks compose buy + sell + validate + emit as a single atomic transaction. If any step fails (price moved, spread too low), the entire transaction reverts with zero cost beyond gas.

Zero-gas price reads. Both price feeds use gasless methods: DeepBook via devInspectTransactionBlock (simulated execution), Cetus via direct pool object reads. The agent scans continuously without spending SUI.

Flash loan arbitrage. The builder supports a capital-free mode: borrow from DeepBook v3 via flash loan, swap on Cetus, repay the loan, keep the profit. All within a single PTB, so the loan is guaranteed to be repaid or the entire transaction reverts.

Integer-only math. All price calculations use bigint (TypeScript) and u128 (Move) to avoid floating-point precision issues. Cetus sqrt prices (X64 fixed-point, 2^64 scaling) are converted via pure integer arithmetic with Newton's method.

Dual monitoring. The listener combines WebSocket event subscriptions (low latency) with polling fallback (reliability). Events are debounced to prevent redundant price checks during high-activity periods.
