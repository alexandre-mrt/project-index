Network Dashboard

Editorial-design personal network dashboard with 142 persona cards, opportunity tracker, and professional situation overview

TypeScript Next.js 15 Supabase Tailwind CSS 4 Biome Bun Vercel

The Challenge

Managing 142+ professional contacts across tiers, tags, and open loops is impossible with spreadsheets or CRMs.

Traditional CRMs are designed for sales funnels, not personal relationship intelligence. You need to track reliability scores, contact freshness, open loops (pending follow-ups), and opportunities linked to specific people, all in a design that feels like reading a magazine, not filling out forms.

This dashboard treats your network as editorial content. Inspired by Monocle magazine's clean aesthetic, it presents persona cards, tier hierarchies, and opportunity pipelines in a format optimized for quick scanning and decision-making. Every contact has a slug-based detail page, reliability dots, tier badges, and a full-text content section parsed from Markdown source files.

Why It Matters

  • Editorial design language: Playfair Display serif headers, parchment backgrounds, forest/sage color palette. No gradients, no rounded corners beyond 4px, no shadows. Reads like a printed publication.
  • 5-tier relationship model: inner-circle, close, active, familiar, acquaintance. Each tier has a distinct badge, filterable in the persona list.
  • Opportunity pipeline: Kanban-style board (active / deadline / standby / completed) with CRUD operations, linked to related personas by ID.
  • Password-protected: simple auth gate via API route and cookie, keeping persona data private without heavy auth infrastructure.

Key Insights

Editorial Design System

Every component follows Monocle magazine aesthetics. Parchment (#F7F6F1) backgrounds, ink (#1A1A1A) text, forest (#2D6A4F) and sage (#8B9D83) accents. No gradients, no rounded corners beyond 4px, no box shadows. The result feels like a printed publication, not a web app.

Tier-Based Relationship Model

Contacts are organized into 5 tiers: inner-circle (closest allies), close, active, familiar, and acquaintance. The PersonaList component supports filtering by tier, by 8 preset tags (sui, bsa, epfl, hackathon, blockchain, crypto, events, kryptosphere), and by free-text search. Sorting works on last contact date, reliability score, name, or tier rank.

Immutable Types Throughout

All interfaces use readonly properties. The Database type maps Supabase tables with strict Row/Insert/Update generics, preventing accidental mutation of ID or timestamp fields. Opportunity updates go through PATCH API routes, never direct state mutation.

Markdown as Source of Truth

The 142 persona cards originate as Markdown files in the Claude Jarvis network directory. The import script parses frontmatter, extracts structured sections, and upserts into Supabase by slug. This means persona data can be edited in any text editor or by Claude Code, then synced to the dashboard with a single bun run import command.
