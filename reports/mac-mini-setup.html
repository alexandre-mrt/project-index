Mac Mini Setup

Automated setup scripts for a 24/7 headless Claude Code server with iMessage Channels

Bash/Zsh macOS Claude Code iMessage LaunchAgent SyncThing tmux SSH

The Challenge

Running a personal AI assistant 24/7 requires careful system engineering.

A Mac Mini sitting on a shelf needs to survive power failures, auto-login without a keyboard, keep Claude Code running across reboots, relay iMessages from family members, and stay in sync with a laptop. Setting this up manually is error-prone and hard to reproduce.

The project automates the entire provisioning of a headless Mac Mini as a Claude Code server. One SSH command from a laptop deploys everything: Homebrew, Bun, Claude Code, tmux sessions, LaunchAgent auto-restart, SyncThing config sync, and iMessage channel routing with per-user context files.

Why It Matters

  • One-command deployment: ./scripts/05-deploy-all.sh mac-mini.local bootstraps the entire server via SSH, from Xcode CLI Tools to a running Claude Channels session.
  • Self-healing: a macOS LaunchAgent restarts Claude Code on boot, after crashes, and daily at 4:00 AM. Power management is configured for zero sleep and auto-restart on power failure.
  • Family-friendly: family members text Claude via iMessage. Each person gets a context file (~/contexts/papa/CONTEXT.md) that controls language, permissions, tone, and capabilities.
  • Config sync: SyncThing keeps skills, commands, settings.json, and CLAUDE.md in sync between the laptop and Mac Mini, while ignoring machine-specific files (history, memory, channels state).

Key Insights

Selective SyncThing Ignoring

Not everything should sync between machines. The .stignore file carefully separates shared config (skills, commands, settings.json, CLAUDE.md) from machine-specific state (history.jsonl, memory/, channels/, projects/). This prevents conflicts and keeps each machine's local context intact.

LaunchAgent vs Cron

macOS LaunchAgents are preferred over cron because they support RunAtLoad (start on boot/login), integrate with launchd's process management, and can set environment variables cleanly. The daily 4 AM restart ensures Claude Code gets a fresh context window and picks up any SyncThing-delivered config changes.

Per-User Context Routing

Each family member gets a dedicated CONTEXT.md file that controls how Claude behaves. This is a simple but effective way to implement multi-tenant access control on a single Claude Code instance. Alexandre gets full technical responses and shell access. Parents get simple French responses with no code execution.

Power Resilience

The bootstrap script configures macOS for maximum uptime: pmset sleep 0, disablesleep 1, displaysleep 0, autorestart 1. Combined with auto-login and LaunchAgent RunAtLoad, the system self-recovers from power outages without any human intervention.
