The Challenge

Your Roomba maps your home every day. It knows the layout of your rooms, where you spend your time, when you're away. That data sits on a corporation's cloud server: unverifiable, mutable, and out of your control.

The core problem is trust in IoT data. There is no way to prove that robot sensor recordings have not been altered after the fact. An insurance company cannot trust a robot's cleaning log. A homeowner cannot prove their robot was operating normally before an incident. A fleet manager cannot audit autonomous behavior with certainty.

RoombaBox makes robot data tamper-proof. It captures sensor snapshots locally, anchors cryptographic hashes on-chain, and stores raw data on decentralized storage. If even a single byte is altered, verification fails. Guaranteed.

Why It Matters

Use Cases

  • Insurance claims — Prove a robot vacuum was operating within normal parameters before a reported incident. Immutable sensor logs serve as evidence.
  • Fleet management — Operators of commercial cleaning robots can audit every session with cryptographic guarantees. No data can be silently deleted or modified.
  • Consumer data sovereignty — Users own their robot's data on decentralized storage instead of trusting a vendor's cloud. Data persists even if the company shuts down.
  • Regulatory compliance — As autonomous robots enter homes and workplaces, regulators will demand auditable behavior logs. RoombaBox provides that infrastructure.

Who Benefits

Homeowners who want control over their IoT data. Insurance companies that need verifiable robot behavior logs. Robot manufacturers who want to offer trust-by-default. Regulators building frameworks for autonomous device accountability.

Data Flow: Capture, Seal, Verify

1. Capture — The edge agent runs a Roomba simulator at 10Hz, producing rich sensor snapshots: position (x,y), heading, speed, battery level, left/right bumper contacts, four cliff sensors, dirt detection, charging state, cleaning mode, and coverage percentage.

2. Seal — Every 30 seconds, the agent collects the snapshot buffer, serializes it with deterministic sorted-key JSON, computes a SHA-256 hash, uploads raw data to Walrus (decentralized blob storage), and anchors the hash on Sui via the BlackBox smart contract. Once anchored, the record is immutable.

3. Verify — Anyone can download the blob from Walrus, recompute the SHA-256 hash, and compare it against the on-chain anchor. If even one byte was altered, the hashes diverge and tampering is detected.

Tamper Detection Demo

A standalone script (demo-tamper.ts) proves the integrity pipeline end-to-end: generate 50 mock snapshots, upload to Walrus, download and verify (PASS), flip a single byte, re-verify (FAIL). The entire demo runs in about 10 seconds.

Smart Contracts (Sui Move)

Three Modules

  • robot_nft — Mints a RobotNFT representing a physical robot. Stores name, model, owner, and creation timestamp. Serves as the authorization key for all BlackBox operations.
  • blackbox — The core contract. A shared BlackBox object linked to a RobotNFT. Manages sessions (start/close), stores AnchorEntry records (data_hash + walrus_blob_id + timestamp), and provides on-chain verify_hash(). Only the NFT holder can write.
  • session_registry — Global registry mapping robot IDs to their BlackBox addresses. Initialized on deploy as a shared object with a Table<ID, address>.

Roomba Simulator

The edge agent includes a full physics-based Roomba simulator with a state machine implementing five cleaning modes:

  SPIRAL --> WALL_FOLLOW --> RANDOM_BOUNCE --> CROSS_ROOM --> SPIRAL (repeat)
                                                    |
                                              (low battery)
                                                    v
                                            RETURN_TO_DOCK --> CHARGING --> SPIRAL
  • Spiral — Outward spiral from current position, growing radius at 0.02m per revolution
  • Wall follow — Right-hand rule along walls and obstacles for up to 30 seconds
  • Random bounce — Straight lines with 120-180 degree bounces on collision
  • Cross room — Direct navigation to a random unvisited area at 0.5 m/s
  • Return to dock — Triggered below 15% battery, navigates to dock position

Coverage is tracked on a 10cm grid. Collision detection handles both rectangular and circular obstacles. 14 sensor channels are emitted per snapshot.

Dashboard

A React + Vite frontend with @mysten/dapp-kit for wallet integration. Features include:

  • Live map — Real-time Roomba position via WebSocket from the edge agent
  • Session list — Browse on-chain sessions fetched from the BlackBox object
  • Integrity verification — Select any session, click "Verify", and the dashboard downloads the Walrus blob, recomputes SHA-256, and compares with the on-chain hash. Displays VERIFIED or TAMPERED with full hash details.
  • Wallet connect — Standard Sui wallet connection for on-chain reads

Key Insights

Walrus + Sui is a natural fit for IoT data integrity. Walrus handles the volume (raw sensor blobs at high frequency) while Sui handles the trust (immutable hash anchors with sub-second finality). The separation means you never put large payloads on-chain, but you still get cryptographic guarantees over every byte.

Deterministic serialization is critical. The hasher uses sorted-key JSON serialization so that identical sensor data always produces identical bytes, and therefore identical SHA-256 hashes. Without this, verification would be non-deterministic and useless.

NFT-gated access control is elegant. The RobotNFT serves as both identity and authorization. Only the holder of the NFT linked to a BlackBox can write anchor entries or close sessions. Ownership transfer is just a standard Sui object transfer.

The simulator is production-grade. Five cleaning modes with realistic physics, collision detection against walls and obstacles, coverage tracking on a 10cm grid, battery management with dock return, and 14 sensor channels. This is not a toy mock; it produces data that closely mirrors real Roomba behavior.

End-to-end tamper detection in 10 seconds. The demo-tamper script proves the entire pipeline: generate data, upload to Walrus, verify (PASS), flip one byte, re-verify (FAIL). A compelling, runnable proof of concept that requires no setup beyond Bun and network access.
