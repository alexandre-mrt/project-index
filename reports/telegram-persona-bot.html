Telegram Persona Bot

Reads Telegram messages via MTProto and generates structured persona cards per contact using Claude API

TypeScript Bun GramJS / MTProto Claude API Privacy-First Biome

The Challenge

Personal networks grow faster than memory can track.

Hundreds of Telegram conversations across private chats and groups contain valuable context about contacts: their interests, expertise, communication style, and relationship history. Manually maintaining a network database is unsustainable, and raw message data is a privacy liability.

Building meaningful professional and personal relationships requires remembering context: what someone works on, what you discussed last, which groups you share, and how often you interact. This information exists scattered across 18 months of Telegram history, buried in thousands of messages across private chats and groups. Extracting it by hand is a full-time job that no one does.

Why It Matters

  • Automated relationship intelligence: transforms raw Telegram messages into structured persona cards with interests, traits, communication style, and relationship context.
  • Privacy by design: raw messages exist only in memory during processing and are aggressively wiped after analysis. No message text is ever persisted to disk.
  • Dual analysis modes: either call Claude API directly (fast, online) or save prompts for batch processing by Claude Code agents (offline, parallelizable).
  • Searchable network index: 142 persona cards indexed with frontmatter metadata (tier, tags, expertise, location) for instant querying by need.

Key Insights

Privacy as Architecture

Raw messages never touch the filesystem. The wipeMessages() function scrubs every message text to an empty string and clears the array to length 0. The withPrivacy() wrapper uses try/finally to guarantee cleanup even when analysis throws. This is the one intentional mutation in an otherwise immutable codebase.

Dual Analysis Modes

In api mode, the bot calls Claude API directly for each contact and writes persona files immediately. In agent mode, it saves structured prompts as .prompt.txt files for batch processing by Claude Code agents, enabling parallelism and offline analysis of the entire network at once.

Resilient Fetching

Each dialog fetch is wrapped in a 20-second AbortController timeout. FloodWait errors trigger exponential backoff (max 120s). Failed dialogs retry twice with 10s delays and automatic reconnection. This makes the bot robust against Telegram rate limits during large-scale fetches.

Tier-Based Network Model

Personas are classified into 5 canonical tiers: inner-circle, close, active, familiar, acquaintance. Non-standard tiers from Claude analysis are auto-normalized (e.g., "weak" becomes "acquaintance", "friend" becomes "close"). The indexer outputs tier distribution and top tags for network overview.

Merge-Over-Overwrite

When a persona already exists on disk, new analysis data is merged into the existing file rather than replacing it. This preserves manually added notes while enriching the card with fresh data from recent messages.
