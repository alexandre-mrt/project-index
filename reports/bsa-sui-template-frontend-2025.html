The Challenge

Building a Sui dApp from scratch means wiring up wallet connection, SDK configuration, Move contract interaction, and decentralized storage individually. There is no good full-stack starter that combines all of these into a working template. Developers spend hours on boilerplate before writing any business logic.

This template solves the cold-start problem: clone, install, run, and you have a working Sui dApp with wallet auth, on-chain transactions, decentralized storage, and identity-based encryption out of the box.

Why It Matters

  • Developer onboarding — new Sui developers can start building features immediately instead of fighting SDK setup
  • BSA 2025 hackathon — official template for the Blockchain Student Association hackathon at EPFL
  • Walrus showcase — demonstrates both free publisher API and paid SDK upload flows for decentralized blob storage
  • Seal integration — working example of identity-based encryption with whitelist access control using Mysten's Seal protocol

Move Contracts

Counter (startHack::counter)

  • Counter shared object with id, owner, value
  • OwnerCap ownership capability transferred on creation
  • create() — anyone can create and share a counter
  • increment(clock, counter) — public, +1 with timestamp event
  • set_value(counter, value, ctx) — owner-only arbitrary set
  • freeze_counter(cap, counter) — permanently freeze

Whitelist (startHack::whitelist)

  • Admin-capped whitelist with versioned shared object
  • create_whitelist() returns (Cap, Whitelist)
  • add() / remove() — cap-gated address management
  • seal_approve(id, wl, ctx) — Seal entry point, checks prefix + membership
  • Key format: [pkg_id][whitelist_id][nonce]

Key Insights

Turbopack by default — Next.js 16 uses Turbopack as the default bundler. No webpack config needed. Fast refresh is near-instant even with heavy SDK imports.

Walrus dual upload pattern

The template demonstrates two upload strategies side by side:

  • Publisher API (free) — simple HTTP PUT to the public testnet publisher. No wallet signature required. Best for hackathons and prototyping.
  • SDK flow (paid, 2 signatures) — writeBlobFlow with register + certify steps. User pays WAL tokens. Production-grade with deletable blobs and ownership tracking.

Seal whitelist encryption

Identity-based encryption via Seal key servers (10 available on testnet). The whitelist Move contract acts as the access control gate: only addresses on the whitelist can call seal_approve, which unlocks decryption keys. Session keys with 10-minute TTL avoid repeated wallet signatures.

Network-aware constants

Package IDs and Walrus endpoints stored per network in constants.ts. The useNetworkVariable() hook resolves the correct value based on active network. Testnet is fully configured; devnet and mainnet are 0xTODO placeholders.
