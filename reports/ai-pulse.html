AI Pulse

Automated AI/tech content pipeline from aggregation to multi-platform publishing

TypeScript Bun Claude API ElevenLabs FFmpeg SQLite Beehiiv Twitter/X YouTube

The Challenge

AI/tech news moves too fast for manual curation.

Hundreds of articles, papers, and product launches drop daily across Hacker News, Reddit, ArXiv, TechCrunch, Product Hunt, and RSS feeds. Manually reading, filtering, writing, and publishing content across newsletter, Twitter, and YouTube is unsustainable.

Staying relevant in the AI space requires covering breaking news within hours. A human curator would need to monitor 6+ sources daily, score articles by relevance, write a newsletter, compose a Twitter thread, script a YouTube video, generate TTS audio, assemble the video, and publish to 3 platforms. That is a full-time job for content that has a 24-hour shelf life.

Why It Matters

  • End-to-end automation: from raw RSS/API feeds to published content on 3 platforms, zero manual intervention required.
  • Multi-source intelligence: 6 aggregators with weighted relevance scoring ensure nothing important is missed and low-quality noise is filtered out.
  • AI-native content: Claude API generates platform-specific content (long-form newsletter, punchy threads, narrated video scripts) from the same curated digest.
  • Faceless YouTube: fully automated video production using ElevenLabs TTS and FFmpeg, no camera or editing software needed.

Scoring formula: combined = relevance * 0.6 + trending * 0.4

Relevance is computed by matching article text against 20 AI/ML keywords. Trending is the normalized engagement score weighted by source credibility. Articles are deduplicated by URL both within a run and across a 7-day window via SQLite.

Key Insights

Parallel Everything

The pipeline maximizes parallelism at every stage. All 6 source providers fetch concurrently via Promise.all. All 3 content types generate in parallel. All 3 publishers run via Promise.allSettled so one failure does not block the others.

Graceful Degradation

Each pipeline stage is wrapped in a runStage helper that catches errors and records status. Publishing uses Promise.allSettled, so if Twitter fails, the newsletter and YouTube video still publish. The final status is success, partial, or failed depending on which stages completed.

Cross-Day Deduplication

Articles are deduplicated not just within a single run, but across a rolling 7-day window stored in SQLite. This prevents the same trending story from appearing in consecutive daily digests.

Faceless Video Production

The YouTube pipeline is fully automated: Claude writes timed narration scripts, ElevenLabs converts each section to speech, and FFmpeg assembles a 1080p video with text overlays synced to audio segments. No camera, no editing, no human in the loop.

Immutable Pipeline State

All pipeline types use readonly arrays and properties. Stage updates produce new arrays via .map() rather than mutating in place. This makes the pipeline run traceable and debuggable at every step.
