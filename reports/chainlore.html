ChainLore

Interactive visual encyclopedia of blockchain history with timelines, force graphs, and full-text search

TypeScript Next.js 16 React 19 D3.js Framer Motion Fuse.js Tailwind CSS 4 shadcn/ui JSON Data

The Challenge

Blockchain history is fragmented across thousands of sources with no unified narrative.

The crypto space spans 15+ years, hundreds of projects, dozens of major hacks, regulatory events, and market cycles. Knowledge lives in scattered blog posts, Twitter threads, and court documents. There is no single place to explore how Bitcoin's genesis connects to DeFi Summer, how the FTX collapse rippled through the ecosystem, or who the key people behind each era were.

ChainLore organizes this history into a structured, browsable encyclopedia. Every project, person, event, and era is a JSON entity with typed fields (funding rounds, token data, key events, related entities). Three visualization modes let users explore the data: a scroll-driven animated timeline, a D3 force-directed relationship graph, and a fuzzy full-text search powered by Fuse.js.

Why It Matters

  • Structured data, not prose: 245 JSON entities with typed schemas (Project, Person, ChainEvent, Era) enable programmatic queries and cross-referencing that plain articles cannot.
  • Visual exploration: a scroll-driven timeline shows events chronologically, while the force graph reveals hidden connections between founders, projects, and events.
  • 10 blockchain eras: from the Genesis era (2008-2011) through ICO Mania, DeFi Summer, NFT Boom, and into the AI-Crypto era (2024+), each with market context and cultural vibe.
  • Full-text search: Fuse.js indexes all entities by name, description, and key fields, returning ranked results with relevance scores.

Key Insights

JSON-First Data Architecture

No database, no CMS, no API. All 245 entities are plain JSON files loaded directly from the filesystem. This makes the data version-controllable, diffable, and trivially extensible. Adding a new project means creating one JSON file with the right schema. The loadEntities function reads an entire directory, parses all JSON files, and returns typed arrays.

Force Graph Relationship Discovery

The D3 force graph builds links from three relationship types: project founders (project -> person), event participants (event -> project, event -> person), and related projects (project -> project). Node radius scales with connection count, making influential entities like Vitalik Buterin or Ethereum visually prominent. The graph supports drag, zoom, and hover tooltips.

Era-Based Timeline Segmentation

All entities are assigned an era (one of 10 defined periods). The timeline view groups nodes by era, with scroll-driven Framer Motion animations revealing each section as the user scrolls. People who do not have an explicit era get one resolved from their earliest key event date via the resolvePersonEra function.

Rich Entity Schemas

Each entity type carries structured metadata beyond basic descriptions. Projects have funding rounds with investors and valuations, token info with ATH prices, and GitHub metrics. People have project roles with time periods, notable quotes, and controversies. Events track financial impact (amount_lost), aftermath descriptions, and cross-references to involved projects and people.
