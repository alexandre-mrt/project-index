SuiPilot

AI-powered Chrome extension wallet for the Sui blockchain with natural language DeFi interactions

TypeScript React 19 Claude API Sui SDK Zustand Chrome MV3 Vite Tailwind CSS 4 Hono Radix UI

The Challenge

Crypto wallets are unusable for most people.

Swapping tokens on a DEX requires understanding slippage, pool selection, and transaction signing. Lending on NAVI or providing liquidity on Cetus means navigating protocol-specific UIs. Even checking a portfolio's performance across DeFi positions demands visiting multiple dApps. The cognitive barrier to entry is enormous.

SuiPilot replaces the traditional wallet UI with a conversational AI agent. Users type "swap 10 SUI for USDC" or "lend 100 USDC on NAVI" in natural language, and Claude builds, simulates, and executes the transaction. Read-only queries like balance checks and price lookups auto-execute instantly, while state-modifying operations (sends, swaps, stakes) require explicit user confirmation before signing.

Why It Matters

  • Natural language DeFi: Claude understands intent and maps it to the right protocol call. No need to learn each dApp's interface.
  • Tool-use architecture: 7 agent tools (balance, prices, portfolio, history, send, swap, stake) give the AI precise on-chain capabilities without hallucinating transactions.
  • Security-first keystore: AES-256-GCM encryption with PBKDF2 (600K iterations), auto-lock after 5 minutes, BIP39 mnemonic with Ed25519 derivation at m/44'/784'.
  • Multi-protocol DeFi: Cetus DEX aggregator for optimal swap routing, NAVI Protocol for lending/borrowing, native SUI staking.

Agent Tools

check_balanceReadFetch SUI and token balances for active account

send_tokensWriteTransfer SUI or tokens to an address

Read tools auto-execute. Write tools (send_tokens, swap_tokens, stake_sui) always trigger a confirmation modal before signing. The agent loops up to 10 tool iterations per message, and conversation history is capped at 50 messages.

Key Insights

Tool-Use Agent over Prompt Engineering

The agent uses Claude's native tool-use capability rather than parsing free-text responses. Each tool has a typed JSON schema (amount, token, address), so the AI cannot hallucinate transaction parameters. The tool handler validates inputs before building any Transaction object.

Confirmation Gate for Write Operations

All state-modifying tools (send_tokens, swap_tokens, stake_sui) are classified as write operations. When the agent calls one, a modal shows the exact parameters (recipient, amount, estimated output) and the user must explicitly confirm. Read tools like check_balance execute silently.

Direct Move Calls for NAVI

The official navi-sdk depends on @mysten/sui v1, which conflicts with this project's v2. Instead of downgrading, SuiPilot extracts NAVI's pool configs and builds raw Move call transactions directly via Transaction.moveCall(). This keeps the dependency tree clean while maintaining full lending/borrowing functionality.

Proxy Architecture

Chrome extensions cannot make direct API calls to Anthropic (CORS restrictions + API key exposure). A lightweight Hono server running on localhost:3001 forwards requests, keeping the API key on the server side. The proxy also handles streaming responses for real-time chat output.
