Whisper Dictate

100% local voice-to-text for macOS, powered by Whisper on Apple Silicon

Python MLX Whisper Apple Silicon macOS APIs PyAudio Quartz AppKit

The Challenge

Cloud-based dictation sends every word you say to external servers.

Slow round-trips, privacy exposure, and zero functionality without an internet connection. Your voice data lives on someone else's infrastructure.

macOS built-in dictation and services like Google Voice require a network connection and route audio through cloud APIs. For developers, writers, and anyone handling sensitive information, this is a non-starter. Latency varies with connection quality, and there is no guarantee about how voice data is stored or processed server-side.

The goal: hold a key, speak, release, and have the transcribed text appear instantly at your cursor. No network. No cloud. No data leaving your machine.

Why It Matters

  • Privacy by design -- audio never leaves your Mac. No server, no API key, no data retention policy to worry about.
  • Apple Silicon optimization -- MLX Whisper runs inference on the Apple Neural Engine, making transcription faster than real-time on M-series chips.
  • Works offline -- airplane mode, no WiFi, VPN down. Dictation keeps working because the model runs locally.
  • System-wide -- global hotkey works in any app. Hold fn in your IDE, browser, or terminal and speak.

Two Operating Modes

Terminal mode uses raw stdin key detection with no special permissions. Global mode uses either a Quartz event tap (for the fn/Globe key, detecting the NSEventModifierFlagFunction flag bit) or pynput keyboard listeners (for all other keys).

Overlay UI

A native AppKit floating window appears at the top-center of the screen while recording. Borderless, translucent, with a pulsing red dot and "Listening..." label. It floats above all windows, ignores mouse events, and joins all Spaces. A background thread watches for parent process death to auto-terminate.

Key Insights

Apple Silicon Acceleration

MLX Whisper runs on the Apple Neural Engine, not just CPU. The mlx-community models are quantized and optimized for M-series chips. The whisper-small-mlx model (~500 MB) transcribes faster than real-time, typically processing 5 seconds of audio in under 1 second.

The fn Key Problem

The fn/Globe key on Mac keyboards is not a regular key. It does not generate standard key press/release events. pynput cannot reliably detect it. The solution: a Quartz CGEventTap that monitors kCGEventFlagsChanged events and checks the 0x800000 (NX_SECONDARYFNMASK) flag bit. This gives true hold-to-talk behavior for the most natural hotkey on a Mac.

CoreAudio Threading Constraint

macOS CoreAudio requires the audio stream to be opened on the main thread. The recorder opens the PyAudio stream on the calling thread, then spawns a background thread for the read loop. Violating this causes silent failures or crashes on macOS.

Hotkey-Driven UX

No GUI, no menubar app, no tray icon. The entire interaction is: hold a key, speak, release. Audio feedback (start/stop sounds) and a pulsing overlay provide confirmation. Text appears at the cursor via clipboard injection (Cmd+V simulated through osascript). A trailing space is appended by default for continuous dictation.

Dual Backend Support

MLX is the primary backend for Apple Silicon. For non-Apple hardware, faster-whisper provides CUDA GPU acceleration or CPU fallback with int8 quantization. Backend selection is a single config toggle.

Model Flexibility

Six model sizes from tiny (75 MB) to large-v3-turbo (1.6 GB). Models are downloaded from HuggingFace on first run or pre-downloaded locally via the interactive download-model.sh script. Supports 20+ languages with auto-detection, plus speech translation to English.
